# Security Policy

## Reporting a Vulnerability

If you find a security vulnerability in Diagraw, please report it privately so
it can be addressed before public disclosure. Do not open a public issue for
security problems.

There are two ways to report:

1. Open a private security advisory on GitHub:
   https://github.com/barmoshe/diagraw/security/advisories/new
2. Email Barprojectsandbuilds@gmail.com

Please include enough detail to reproduce the issue: affected files or routes, a
description of the impact, and steps to trigger it. If you have a suggested fix,
that is welcome but not required.

You can expect an acknowledgement of your report, and we will keep you updated
as the issue is investigated and resolved.

## Supported Versions

Diagraw is pre-1.0 and moving quickly, so there are no formal supported-version
guarantees yet. Please report issues against the current `main` branch. Once the
project reaches a stable 1.0 release, this policy will be updated with a
supported-version table.
